# CV Handoff — Epsilon Quant Research (Justin Iturregui)

> Hub: [[COWORK]] · [[STRATEGY_REFERENCE]] · [[POLYMARKET_BRAIN]]

## Project overview

**Epsilon** is a personal quant research operation — two parallel tracks, one repo:

1. **Crypto live trading** — systematic momentum / stat-arb / BB-breakout strategies running live on Binance futures, supported by a custom research and backtesting stack.
2. **Polymarket alpha** — prediction-market trading research (options-delta pricing, market-making, copy-trading) culminating in a live execution bot.

---

## 1. Research & backtesting infrastructure (crypto)

Built from scratch. Nothing off-the-shelf.

- **Walk-forward engine** (`infrastructure/walkforward/wf_engine.py`) — rolling in-sample/out-of-sample optimisation with Optuna (TPE sampler). Prevents look-ahead by design. Used as the inner loop for CPCV.
- **CPCV engine** (`infrastructure/walkforward/cpcv_engine.py`) — Combinatorial Purged Cross-Validation, a more rigorous backtest framework than standard CV (avoids distribution leakage). Emits Pearson r CIs, not just point estimates. Covers 7 assets per strategy family.
- **Backtester** (`infrastructure/backtester/engine.py`) — single-asset simulation engine with full performance metrics (Sharpe, drawdown, CAGR), portfolio aggregation, and ML metrics.
- **Data pipeline** — Binance OHLCV ingestion (hourly + daily), cached as Parquet, with a backfill utility and a cache manager. Universe: BTC, ETH, SOL, ADA, XRP, AVAX, LINK, ATOM, ARB, APT, NEAR, SNX, FIL, TRX, LTC, POL.
- **Cross-sectional strategy factory** (`infrastructure/walkforward/xs_strategy.py`) — framework for cross-sectional momentum research across multiple assets simultaneously.

Tech: Python ≥3.10, Pandas, NumPy, Optuna, SciPy, PyArrow/Parquet, Jupyter notebooks.

## 2. Crypto strategies researched and deployed

Three strategy families, all with walk-forward / CPCV validation:

### Momentum (live)
Two production variants:
- `momentum_swing` — volume-MA filtered momentum (ETH, SOL, ADA, XRP, AVAX)
- `momentum_no_vol` — no volume filter (BTC, AVAX)

Live universe: ADA, AVAX, BTC, ETH, SOL, XRP. Validated per-asset via CPCV notebooks; research in `topics/momentum/strategies/momentum_cpcv/`.

### Bollinger Band breakout (research, not yet live)
Full CPCV validation across 7 assets (ADA, AVAX, BTC, ETH, LINK, MATIC, NEAR). Parameters tuned via walk-forward; results staged in `topics/momentum/strategies/bb_cpcv/`.

### Statistical arbitrage / pairs trading (research)
Rolling OLS spread → z-score → mean-reversion signals. Entry/exit on z-score bands with stop and max-holding controls. Live dashboard stub wired up (`live_trading/dashboards/statarb/`).

### Cross-sectional momentum (in progress)
Multi-asset rank-based momentum using `xs_strategy.py`. Not yet productionised.

## 3. Live trading system

A Streamlit-based trading dashboard suite wired to live Binance data:

- **Unified app** (`live_trading/app.py`) — multi-page Streamlit app: live signals + trade entry forms, trade journal, portfolio equity/drawdown charts.
- **Real-time data** — WebSocket manager for live price feeds; hourly/daily Parquet cache updated on a schedule.
- **Signal engine** — live parameter files (`live_params.json`) generated by the walk-forward optimiser per asset; signals recomputed each render cycle.
- **Trade logging** — append-only `trades.json` per strategy; positions read fresh every cycle.
- **Portfolio view** — equity curve and drawdown charts across all open strategies.

Tech: Streamlit, Plotly, python-binance, Optuna (live re-optimisation).

## 4. Polymarket research (options-delta, market-making)

~9 months of systematic alpha research on Polymarket prediction markets. Highlights:

- **Options-delta pricing (Block K / OD thread)** — treated Polymarket binary contracts as digital options; built a vol-mispricing + delta-hedge framework (Strategy A) using Binance crypto-4h prices as the underlying. Ran full in-sample / OOS / embargoed backtests with confidence intervals. Falsified several branches on robust grounds (non-overlap math, real adverse selection), preserved others. Key findings in `polymarket/research/notes/options_delta/`.
- **Market-making research (MM thread)** — analysed Polymarket maker economics at depth (quoting, fill rates, real-maker moat via K5 wallet-level data). Concluded MM sub-scale standalone; reframed MM as the execution layer for OD.
- **TOB/OFI microstructure (dali lineage)** — 17-block research arc testing top-of-book order-flow imbalance as a short-horizon signal. Multi-tier testing: taker entry, maker entry, LightGBM ensemble. All falsified under non-overlap math; documented as publishable Polymarket microstructure study.
- **DuckDB + Parquet data infra** — all research uses DuckDB over Parquet shards (append-only). Lookahead-free metric computation enforced by design. Managed via uv.

## 5. Polymarket live trading bot (Midas)

A production-grade copy-trading execution bot for Polymarket:

- **Signal source** — Polymarket RTDS WebSocket (`wss://ws-live-data.polymarket.com`) filtered client-side on a leader's proxy wallet address. Reconnect-gap fallback via REST data API.
- **Execution layer** — full async execution pipeline: watcher → signal → risk → mirror engine → venue adapter → Polymarket CLOB (via py-clob-client + custom HTTP client). Real-venue adapter handles order signing, submission, and fill journaling.
- **Risk management** — per-trade USD cap, per-market cap, total-deployed cap, price-deviation guard, daily-loss halt, file-presence kill switch, max open positions.
- **Copy logic** — mirrors one leader's fills proportionally (fixed USD sizing for PoC: $50–$100 real money). Leader quality determined offline via research; execution layer has no leader-health logic by design.
- **Testing** — pytest suite for executor and harvester packages; WebSocket probe scripts with documented findings.

Tech: Python asyncio, WebSocket, py-clob-client, pytest.
